## What This Does

This is a fully headless Node.js scraper that logs into **myvistage.com** once per run, crawls network activity feeds, extracts member profiles, scores them for CRM / ERP / AI fit, and appends new prospects to a Google Sheet. It also sends keyword-alert emails when posts mention CRM, ERP, or AI topics.

- Runs every **30 minutes**, 2 AM – 11 PM ET
- Respects a **daily quota** of 12 network visits to avoid rate-limiting
- Uses **Playwright + Stealth** to browse as a real user (headless Chromium)
- Writes results to Google Sheets via the **Google Sheets API** (OAuth2)
- Sends email alerts via **Nodemailer / SMTP**
- Persists scraper state in `state.json` and browser cookies in `browser-state.json`

After 838+ prospects collected, it continues in **deep scan mode** (120-day lookback) to catch anyone new.

## Proxmox VM Requirements

### Recommended VM specs
- **OS:** Ubuntu 22.04 LTS or Debian 12 (minimal install is fine)
- **CPU:** 2 vCPUs
- **RAM:** 2 GB (Playwright Chromium needs ~300–500 MB per run)
- **Disk:** 10 GB
- **Network:** Must have outbound internet access on ports 80, 443

### Software to install on the VM
```bash
# Node.js 20 LTS
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs

# Verify
node --version   # should be v20.x
npm --version

# Playwright system dependencies (Chromium needs these)
sudo apt-get install -y \
  libnss3 libatk1.0-0 libatk-bridge2.0-0 libcups2 \
  libxkbcommon0 libxcomposite1 libxdamage1 libxfixes3 \
  libxrandr2 libgbm1 libasound2 libpango-1.0-0 libcairo2 \
  fonts-liberation wget ca-certificates

# Git
sudo apt-get install -y git
```

## Setup Steps (run in order on the Proxmox VM)

### 1. Clone the repo
```bash
git clone https://github.com/crmexpertsnyc/vistage-prospecting.git
cd vistage-prospecting
```

### 2. Install Node dependencies + Playwright browser
```bash
npm install
npx playwright install chromium
```
> This downloads a ~120 MB Chromium binary. Only needs to run once.

### 3. Transfer secret files from John's computer

Copy these four files into the project directory on the VM.  
**Do NOT put them in git — they are gitignored.**

| What to transfer | How |
|-----------------|-----|
| `.env` | `scp .env user@vm-ip:/path/to/vistage-prospecting/` |
| `credentials.json` | Same via scp |
| `alert-config.json` | Same via scp |
| `token.json` | Generate fresh on the VM (see Step 4) — do NOT copy the Windows one |

**Contents of `.env`** (create from `.env.example`):
```
VISTAGE_EMAIL=jperez@crmexpertsny.com
VISTAGE_PASSWORD=<ask John for the password>
```

### 4. Set up Google OAuth2 (run auth.js ONCE on the VM)

> `token.json` is tied to the machine's OAuth flow — you must generate a fresh one on the VM.
> You need a display / browser for this step. Use SSH with X11 forwarding, or run it in a VM console session.

**Option A — SSH with X11 forwarding (from your local machine):**
```bash
ssh -X user@vm-ip
cd /path/to/vistage-prospecting
node auth.js
# A browser will open on your local screen — sign in with the Google account
# that owns the Google Sheet. token.json will be saved automatically.
```

**Option B — Copy auth URL, open on local browser:**
```bash
# On the VM:
node auth.js
# It prints an auth URL. Copy it, open it in a browser on any machine,
# authorize, then copy the redirect URL back — or just use the printed URL.
```

After `node auth.js` completes you should see:
```
✅ token.json saved successfully.
```

### 5. Test the scraper manually
```bash
node scraper.js
```
Expected output: a run summary like the one below, no errors.
```
Run complete       : 2026-04-21T...
New people         : 0-6
Total all-time     : 838+
Sheet              : https://docs.google.com/spreadsheets/...
```

### 6. Install the cron job
```bash
bash cron-setup.sh
```
This registers a cron job that runs `run.sh` every 30 minutes.  
`run.sh` handles the active-hours gate (2 AM – 11 PM ET) internally.

### 7. Verify cron is running
```bash
crontab -l                          # see the registered job
tail -f run.log                     # watch live output
```

## Google Cloud Console Setup (if credentials.json needs to be recreated)

1. Go to https://console.cloud.google.com/
2. Select (or create) project **vistagescrape**
3. Enable **Google Sheets API** under APIs & Services → Library
4. Go to APIs & Services → Credentials → Create Credentials → OAuth 2.0 Client ID
5. Application type: **Desktop app**, name it `vistage-scraper`
6. Download the JSON → save as `credentials.json` in the project directory
7. Run `node auth.js` to generate `token.json`

2. Go to Google Account → Security → 2-Step Verification → App passwords
3. Create an app password for "Vistage Scraper"
4. Use the 16-character password as `smtp_pass` above

## Key Error Messages

| Message in log | Meaning | Fix |
|---------------|---------|-----|
| `token.json not found` | Google OAuth not set up | Run `node auth.js` |
| `VISTAGE_EMAIL or VISTAGE_PASSWORD not set` | `.env` file missing | Create `.env` from `.env.example` |
| `Could not log in` | Vistage password changed or account locked | Check credentials, re-run scraper manually |
| `CAPTCHA` | Vistage triggered bot detection | Wait 1–2 hours, then retry |
| `Daily quota reached` | Already hit 12 networks today | Normal — resets each run day |

## Architecture Notes

- **State persistence:** `state.json` tracks all discovered networks, visited profiles, processed threads, run counts, and consecutive zero-result counts. Do not delete it — it contains 838+ discovered prospects.
- **Browser state:** `browser-state.json` stores Vistage session cookies so the scraper stays "logged in" across runs without re-authenticating every time.
- **Quota system:** The scraper is intentionally conservative (6 profiles/run, 12 networks/day) to avoid triggering Vistage's rate limits or CAPTCHA.
- **Deep scan mode:** After 3+ consecutive zero-result runs, the scraper automatically expands its lookback window from ~60 days to ~120 days to find older posts.
- **Stealth mode:** Uses `playwright-extra` + `puppeteer-extra-plugin-stealth` to disguise Chromium as a regular browser.
